## Proposed Changes

### 1. Explainable AI (XAI)
#### [NEW] [explain_gat_118bus.py](file:///c:/Users/Avijit%20Baidya/OneDrive/Desktop/Grid_Project/src/explain_gat_118bus.py)
* **Goal**: Provide the grid operator with physical explanations of anomalies.
* **Mechanism**:
  - Load the trained scaler, models, and a test snapshot where an attack occurred on Bus 105 (Snap 013).
  - Enable gradient tracking on the input feature tensor: `x.requires_grad = True`.
  - Pass through the GAT Node Localizer model: `out = model(x, edge_index)`.
  - Compute `loss = out[105]` (the output logit for the attacked bus).
  - Execute a backward pass `loss.backward()` to obtain feature gradients: $\mathbf{G} = \frac{\partial P(\text{Attack})}{\partial \mathbf{x}}$.
  - Compute node feature attributions: $\mathbf{A} = |\mathbf{G}|$.
  - Summarize attributions per feature group (Voltage Magnitude, Voltage Angle, Active Power, Reactive Power, and Connected Line Currents).
  - Save a horizontal bar chart plot as `plots/gat_explanation_bus105.png`.

### 2. Closed-Loop Mitigation & State Reconstitution
#### [NEW] [closed_loop_mitigation.py](file:///c:/Users/Avijit%20Baidya/OneDrive/Desktop/Grid_Project/src/closed_loop_mitigation.py)
* **Goal**: Simulate grid self-healing.
* **Mechanism**:
  - Stream synchrophasor measurements.
  - When the GNN Node Localizer flags a bus (anomaly probability $> 0.5$):
    1. **Quarantine**: Flag the PMU measurements of this bus as "untrusted".
    2. **Reconstitute**: Calculate the true voltage magnitude ($V_i$) and angle ($\theta_i$) using a spatial estimation from adjacent secure neighbor buses:
       $$\hat{V}_i = \frac{1}{|\mathcal{N}_i|} \sum_{j \in \mathcal{N}_i} V_j$$
       $$\hat{\theta}_i = \frac{1}{|\mathcal{N}_i|} \sum_{j \in \mathcal{N}_i} (\theta_j - \delta_{ji})$$
       where $\delta_{ji}$ is the nominal physical phase angle difference between bus $j$ and $i$ derived from the Digital Twin topology.
    3. **Solve State Estimation**: Update measurement inputs with the reconstructed values and calculate state estimation errors before, during, and after mitigation.
  - Generate a plot comparing the state estimation errors and save it to `plots/closed_loop_mitigation_error.png`.

### 3. Multi-Bus Attack Scaling
#### [NEW] [test_multibus_scaling.py](file:///c:/Users/Avijit%20Baidya/OneDrive/Desktop/Grid_Project/src/test_multibus_scaling.py)
* **Goal**: Evaluate GNN detection under scaled adversarial bounds.
* **Mechanism**:
  - Write a script that dynamically perturbs $K \in \{1, 2, 3, 4, 5\}$ randomly selected buses simultaneously in secure snapshots.
  - Coordinate the line current sensors on all adjacent lines to keep the attack stealthy (BDD residual low).
  - Run the trained Graph Classifier and Node Localizer GNNs on these multi-bus attack sets.
  - Measure the recall rates for each scale $K$.
  - Plot GNN recall vs. the number of attacked buses and save it to `plots/multibus_scaling_recall.png`.
